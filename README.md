# 汉字编码定位 Geohanzi

一个纯前端（单文件 `index.html`）的 GPS 定位与坐标编码互转工具，主要面向 iOS Safari，桌面浏览器同样可用。

## 功能

界面分两个 Tab：

- **获取定位**：调用浏览器 Geolocation API。每次点击都固定为「高精度 + 全新定位」（`enableHighAccuracy: true`、`maximumAge: 0`，不复用缓存），失败时自动重试一次，降低用户心智负担。
- **手动输入**：输入框支持粘贴/输入下列任意一种格式，自动识别并反解回坐标，再换算出其余全部格式。

定位或解析成功后，把坐标同时展示为 6 种表示：

1. **汉字编码**（4 字，字库 1024 = 2¹⁰，40 bit）
2. **数字编码**（15 位数字，5 位一组 × 3 组，约 1 米精度）
3. **Geohash**（10 位，50 bit）
4. **Plus Codes**（Open Location Code，8+2 位）
5. **十进制经纬度**（精确到 4 位小数）
6. **度分秒 DMS**（精确到秒）

其它：

- **一键跳转地图**：将坐标转换到对应坐标系后，跳转到高德、百度、苹果地图、Google 地图（GCJ-02 / WGS-84 两版）、Geohash 地图、Google Earth。
- **点击复制**：点击任意结果卡片即可复制，底部弹出提示显示复制的格式与内容。
- 亮/暗色自适应，Apple 风格 UI，无任何外部依赖与网络请求。

## 编码原理

汉字编码、数字编码、Geohash 三者同源——都把经纬度按 **Z 序交织**（interleave，经度优先）成一个大整数，只是最后转成不同进制：

| 编码 | 精度基元 | 进制 | 输出 |
|---|---|---|---|
| 汉字编码 | 40 bit（经 20 + 纬 20） | base-1024（汉字字库） | 4 字 |
| Geohash | 50 bit（经 25 + 纬 25） | base-32（标准字母表） | 10 位 |
| 数字编码 | 见下（十进制网格交织） | base-10 | 15 位数字 |

### 汉字编码

- 字库 `hanzi_freq_1024.txt`：按字频排序的 1024 个常用汉字（去重）。
- 每字承载 log₂(1024) = 10 bit，4 字 = 40 bit。

### 数字编码

15 位十进制正好容纳 10¹⁵ 个格。若沿用二进制交织只能用到 2⁴⁹（约 56%，浪费 15 × log₂10 − 49 ≈ 0.83 bit）。为铺满，改用**十进制网格交织**：每一位把当前格切成 10 个子格并交替细分——

- 偶数位：经度 ÷5、纬度 ÷2（5×2 网格）
- 奇数位：经度 ÷2、纬度 ÷5（2×5 网格）
- 数字值 `d = row × cols + col`（0–9）

累计经度 5⁸·2⁷ = 5×10⁷ 格、纬度 2⁸·5⁷ = 2×10⁷ 格，乘积正好 10¹⁵，首位可达 9、无浪费，精度约 1 米，并保留 Z 序空间局部性（相邻点前缀数字相同）。

### 汉字字库来源

字频数据取自 [Chinese Character Frequency List（MTSU）](http://lingua.mtsu.edu/chinese-computing/statistics/char/list.php?Which=TO)，从中取字频最高的前 1024 个字作为字库。

## 坐标系与地图跳转

中国大陆各地图使用不同坐标系，跳转前会做相应转换（页面上以标签标注）：

| 地图 | 坐标系 | 转换链路 |
|---|---|---|
| 高德地图 | GCJ-02（火星坐标） | WGS-84 → GCJ-02 |
| 苹果地图 | GCJ-02 | WGS-84 → GCJ-02 |
| Google 地图（GCJ-02） | GCJ-02 | WGS-84 → GCJ-02（对齐国内路网/手机端） |
| Google 地图（WGS-84） | WGS-84 | 不转换（对齐卫星影像/电脑网页版） |
| 百度地图 | BD-09 | WGS-84 → GCJ-02 → BD-09 |
| Geohash 地图 | WGS-84 | 不转换 |
| Google Earth | WGS-84 | 不转换 |

高德、百度按环境分流唤端：

- **安卓**：用 `intent://` URL，未装 App 时由系统自动跳网页版（不白屏）。
- **iOS**：用 App scheme 唤端，未装 App 时由超时兜底跳网页版（iOS 无法检测 App 是否安装，可能先闪一下系统弹框）。
- **微信内置浏览器**：私有 scheme 被拦截，直接回落网页版。
- **桌面浏览器**：网页版。

苹果地图、Google 地图、Geohash 地图、Google Earth 均为通用 https 链接，两端通用。

## 使用

直接用浏览器打开 `index.html` 即可，无需构建。

> **注意**：iOS Safari 的定位 API 要求页面通过 **HTTPS 或 localhost** 访问。用 `file://` 或局域网 `http://` 打开会拿不到定位。部署到任意 HTTPS 静态托管即可正常使用。
